## 🇵🇱 Polski

### Samofakturowanie (Self-Billing)

Nowe wydanie wprowadza pełny moduł samofakturowania — od zarządzania dostawcami i umowami, przez generowanie i import faktur, aż po workflow akceptacji przez sprzedawcę.

#### Dostawcy

- Strona dostawców z listą, filtrowaniem po statusie, wyszukiwaniem i paginacją
- Szczegóły dostawcy z danymi kontaktowymi, statystykami i kartą umowy SB
- Tworzenie dostawcy z NIP (weryfikacja Białą Listą)
- Szablony SB per dostawca — CRUD z dialogiem (nazwa, opis pozycji, ilość, jednostka, cena, stawka VAT, waluta)

#### Umowy samofakturowania

- Karta aktywnej umowy z datami obowiązywania i statusem
- Załączniki umów — przesyłanie i lista plików

#### Faktury samofakturowania

- Strona samofakturowania z listą faktur SB, filtrami statusu i wyszukiwaniem
- Generowanie faktur — dialog z wyborem okresu, podglądem i potwierdzeniem
- Import z pliku — pobieranie szablonu CSV/XLSX, przesyłanie pliku, dwuetapowa walidacja
- Workflow statusów: Draft → PendingSeller → SellerApproved → SentToKsef (+ SellerRejected)
- Akcje wierszowe — podgląd, wysyłka do KSeF, akceptacja/odrzucenie z komentarzem

#### Akceptacja faktur SB

- Dialog akceptacji z polem numeru faktury i komentarzem
- Edycja numeru faktury przez sprzedawcę podczas zatwierdzania
- Dialog akceptacji na stronach list `/approvals` i `/self-billing`
- Auto-przełączanie na zakładkę SB gdy brak zwykłych faktur oczekujących

#### Dashboard

- Nowe kafelki KPI: wysłane w bieżącym miesiącu, wygasające umowy (≤30 dni)

### Powiadomienia — ulepszenia

- Scrollowanie panelu powiadomień — poprawiony overflow dla dużej liczby powiadomień
- Filtrowanie: przyciski „Wszystkie" / „Nowe" z badge liczbą nieprzeczytanych
- Oznacz wszystkie jako przeczytane — nowy endpoint `POST /api/notifications/mark-all-read`
- Nowe typy powiadomień: `SbApprovalRequested` (5) i `SbApprovalDecided` (6)

### Poprawki

- Sygnatura komponentu `RowActions` — przyjmuje obiekt faktury zamiast samego ID
- Ekstrakcja komponentów SB do `components/self-billing/` i `components/suppliers/`

## 🇬🇧 English

### Self-Billing

This release introduces a complete self-billing module — from supplier and agreement management, through invoice generation and import, to seller approval workflow.

#### Suppliers

- Suppliers page with list, status filtering, search and pagination
- Supplier detail with contact data, statistics and SB agreement card
- Create supplier from VAT ID (White List verification)
- Per-supplier SB templates — CRUD with dialog (name, item description, quantity, unit, price, VAT rate, currency)

#### Self-Billing Agreements

- Active agreement card with validity dates and status
- Agreement attachments — upload and list files

#### Self-Billing Invoices

- Self-billing page with invoice list, status filters and search
- Invoice generation — dialog with period selection, preview and confirmation
- File import — CSV/XLSX template download, file upload, two-step validation
- Status workflow: Draft → PendingSeller → SellerApproved → SentToKsef (+ SellerRejected)
- Row actions — preview, send to KSeF, approve/reject with comment

#### SB Invoice Approval

- Approval dialog with invoice number field and comment
- Invoice number editing by seller during approval
- Approval dialog on `/approvals` and `/self-billing` list pages
- Auto-switch to SB tab when no regular invoices are pending

#### Dashboard

- New KPI tiles: sent this month, expiring agreements (≤30 days)

### Notifications — Improvements

- Notification panel scrolling — fixed overflow for large number of notifications
- Filtering: "All" / "Unread" toggle buttons with unread count badge
- Mark all as read — new `POST /api/notifications/mark-all-read` endpoint
- New notification types: `SbApprovalRequested` (5) and `SbApprovalDecided` (6)

### Fixes

- `RowActions` component signature — accepts invoice object instead of ID only
- SB component extraction to `components/self-billing/` and `components/suppliers/`
