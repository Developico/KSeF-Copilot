## 🇵🇱 Polski

### Zgodność XML z interpretacją KIS — samofakturowanie

Wersja 0.3.5 implementuje wymagania interpretacji indywidualnej KIS z 27.02.2026 (0112-KDIL1-3.4012.874.2025.2.KK) dotyczące samofakturowania w KSeF. Zgodnie z interpretacją:

- ✅ Sprzedawca musi zatwierdzić fakturę **przed** wysyłką do KSeF
- ✅ Zatwierdzenie musi opierać się na **pliku XML** (nie PDF, nie widoku UI)
- ✅ Korekty XML muszą być możliwe w przypadku zastrzeżeń sprzedawcy
- ✅ Brak prawa do odliczenia VAT jeśli zatwierdzenie nastąpi po wysyłce do KSeF

#### Generowanie i przechowywanie XML

- XML generowany w momencie **składania do akceptacji** (submit), nie przy wysyłce do KSeF — sprzedawca zatwierdza dokładnie ten XML, który trafi do KSeF
- Hash SHA256 obliczany i zapisywany na fakturze (`xmlHash`) — integralność XML weryfikowana na każdym etapie workflow
- Wysyłka do KSeF korzysta z **zapisanego XML** (bez regeneracji), z walidacją hash
- Cofnięcie do Draft kasuje XML i hash (wymusza ponowne wygenerowanie po edycji)

#### Metadane zatwierdzenia w XML (DodatkowyOpis)

- Po zatwierdzeniu przez sprzedawcę XML jest regenerowany z sekcją `DodatkowyOpis` zawierającą:
  - Data zatwierdzenia (ISO timestamp)
  - Osoba zatwierdzająca (email + nazwa)
  - System zatwierdzenia: „KSeF Copilot by Developico"
  - Hash XML przy zatwierdzeniu (SHA256)
- Stopka (`Stopka/StopkaFaktury`) zawiera czytelną notę: „Zatwierdzona przez {name} dnia {date} w systemie KSeF Copilot by Developico"

#### Auto-approve

- Umowy SB z flagą `autoApprove` pomijają etap PendingSeller — faktury automatycznie zatwierdzane przy submicie z pełnym audit trail

#### Podgląd faktury XML

- Nowy komponent `<InvoiceXmlPreview>` — wizualizacja faktury w czytelnym formacie tabelarycznym inspirowanym oficjalnym XSL Ministerstwa Finansów:
  - Nagłówek: kod formularza, wariant, data wytworzenia
  - Sprzedawca (Podmiot1) | Nabywca (Podmiot2) obok siebie
  - Podmiot3 (wystawca) — dla samofakturowania
  - Tabela pozycji: Nr | Nazwa | Ilość | Jm. | Cena netto | Stawka VAT | Netto | Brutto
  - Podsumowanie VAT per stawka + kwota brutto ogółem
  - Adnotacje, warunki płatności, rachunek bankowy
  - DodatkowyOpis (tabela klucz-wartość z danymi zatwierdzenia)
  - Stopka
- Zakładki w widoku szczegółów SB: „Dane" | „Podgląd faktury" | „XML" (surowy)
- Przycisk „Drukuj / Zapisz PDF" — `window.print()` z layoutem A4 (`@media print`)
- Przycisk „Pobierz XML" — endpoint `GET /api/self-billing/invoices/:id/xml`

#### Podgląd XML dla faktur zsynchronizowanych z KSeF

- Reużycie `<InvoiceXmlPreview>` dla istniejącego pola `rawXml` na fakturach pobranych z KSeF
- Sekcja zwijalna w widoku szczegółów faktury (web + code-app)
- Naprawiony mapping pól `rawXml` / `xmlContent` między API a frontendem

#### Infrastruktura Dataverse

- Nowe kolumny na tabeli `dvlp_ksefselfbillinginvoice`:
  - `dvlp_xmlcontent` — Memo (max 1 048 576 znaków) — wygenerowany XML
  - `dvlp_xmlhash` — String (max 100) — SHA256 hash XML
- Skrypt `Provision-SbApprovalColumns.ps1` rozszerzony o automatyczne tworzenie nowych kolumn

#### Nowe klucze i18n

- `xmlPreview.*` (35+ kluczy) w `pl.json` i `en.json` — etykiety podglądu XML (web + code-app)

## 🇬🇧 English

### KIS Interpretation XML Compliance — Self-Billing

Version 0.3.5 implements requirements from the KIS individual tax interpretation dated Feb 27, 2026 (0112-KDIL1-3.4012.874.2025.2.KK) regarding self-billing in KSeF. The interpretation requires:

- ✅ Supplier must approve invoice **before** KSeF submission
- ✅ Approval must be based on the **XML file** (not PDF, not UI view)
- ✅ XML corrections must be possible if supplier raises objections
- ✅ No VAT deduction right if approval happens after KSeF submission

#### XML Generation & Storage

- XML generated at **submit time** (not at KSeF send) — supplier approves exactly the XML that will be sent to KSeF
- SHA256 hash computed and stored on invoice (`xmlHash`) — integrity verified at every workflow stage
- KSeF send uses **stored XML** (no regeneration), with hash validation
- Revert to Draft clears XML and hash (forces regeneration after edits)

#### Approval Metadata in XML (DodatkowyOpis)

- After supplier approval, XML is regenerated with `DodatkowyOpis` section containing:
  - Approval date (ISO timestamp)
  - Approver identity (email + name)
  - Approval system: "KSeF Copilot by Developico"
  - XML hash at approval time (SHA256)
- Footer (`Stopka/StopkaFaktury`) includes readable note: "Approved by {name} on {date} in KSeF Copilot by Developico"

#### Auto-approve

- SB agreements with `autoApprove` flag skip PendingSeller stage — invoices automatically approved on submit with full audit trail

#### Invoice XML Preview

- New `<InvoiceXmlPreview>` component — renders invoice in readable tabular format inspired by official MF XSL:
  - Header: form code, variant, creation date
  - Seller (Podmiot1) | Buyer (Podmiot2) side by side
  - Podmiot3 (issuer) — for self-billing
  - Line items table: Nr | Name | Qty | Unit | Net Price | VAT Rate | Net | Gross
  - VAT summary per rate + gross total
  - Annotations, payment terms, bank account
  - DodatkowyOpis (key-value table with approval data)
  - Footer
- Tabs in SB detail view: "Data" | "Invoice Preview" | "XML" (raw)
- "Print / Save PDF" button — `window.print()` with A4 layout (`@media print`)
- "Download XML" button — endpoint `GET /api/self-billing/invoices/:id/xml`

#### XML Preview for Synced KSeF Invoices

- Reuse of `<InvoiceXmlPreview>` for existing `rawXml` field on KSeF-synced invoices
- Collapsible section in invoice detail view (web + code-app)
- Fixed field name mismatch `rawXml` / `xmlContent` between API and frontend

#### Dataverse Infrastructure

- New columns on `dvlp_ksefselfbillinginvoice` table:
  - `dvlp_xmlcontent` — Memo (max 1,048,576 chars) — generated XML content
  - `dvlp_xmlhash` — String (max 100) — SHA256 hash of XML
- `Provision-SbApprovalColumns.ps1` script extended with automatic column provisioning

#### New i18n Keys

- `xmlPreview.*` (35+ keys) in `pl.json` and `en.json` — XML preview labels (web + code-app)
