# Code Apps — Deployment Guide

Deployment procedure for **KSeF Copilot** as a Power Apps Code App.

Code Apps let you publish a Vite + React SPA directly to Power Platform as a standalone app — visible in `make.powerapps.com`, shareable like canvas apps, and manageable within solutions.

> **Reference**: [Power Apps Code Apps documentation](https://learn.microsoft.com/en-us/power-apps/developer/code-apps/overview)

---

## Prerequisites

### 1. Enable Code Apps in your environment

1. Go to [Power Platform Admin Center](https://admin.powerplatform.microsoft.com/)
2. **Manage** → **Environments** → select your environment
3. **Settings** → **Product** → **Features**
4. Toggle **Power Apps code apps** → **On**
5. Click **Save**

### 2. License

End-users running code apps need a **Power Apps Premium** license.

### 3. Install PAC CLI

```powershell
winget install Microsoft.PowerAppsCLI
```

After installation, restart the terminal and verify:

```powershell
pac --version
```

### 4. Install project dependencies

```powershell
cd code-app
pnpm install
```

---

## One-time setup

### Authenticate and select environment

```powershell
pac auth create
pac env select --environment <environment-ID-or-URL>
```

Sign in with your Power Platform account when prompted. You can find your environment ID in the Admin Center or use the environment URL (e.g. `https://orgXXXXXXXX.crm4.dynamics.com`).

To verify your connection:

```powershell
pac auth list
pac env who
```

### Initialize the Code App

```powershell
pac code init --displayname "KSeF Copilot"
```

This creates a `power.config.json` file in the project root. **Do not modify this file manually** — it's managed by the PAC CLI and Power Apps SDK.

> Optional parameters:
> - `--description "KSeF integration and invoice management"`
> - `--logoPath ./public/logo.png`
> - `--buildPath dist` (defaults to Vite's output)

## Deploy

### Build and push (single command)

```powershell
pnpm run build | pac code push
```

To deploy into a **specific solution**:

```powershell
pnpm run build | pac code push --solutionName "KSeF"
```

If successful, the CLI returns a Power Apps URL where you can run the app.

### Verify deployment

```powershell
pac code list
```

Or open [Power Apps](https://make.powerapps.com/) → **Apps** — the app appears as a standalone application.

---

## Update (subsequent deploys)

Every time you want to publish a new version:

```powershell
pnpm run build | pac code push
```

The same command handles both initial publish and updates.

---

## Add to a solution (ALM)

If you used `--solutionName` during push, the app is already in that solution.

To verify:

1. Go to [make.powerapps.com](https://make.powerapps.com/)
2. **Solutions** → open your solution
3. The code app should appear under **Apps**

The app travels with the solution during export/import across environments.

## Data sources (optional)

Code Apps can use Power Platform connectors directly from JavaScript. To add a data source:

```powershell
# List available connectors
pac code list-tables --apiId shared_commondataserviceforapps

# Add Dataverse as a data source
pac code add-data-source --apiId shared_commondataserviceforapps
```

> **Note**: KSeF Copilot currently connects to its own Azure Functions API (`VITE_API_BASE_URL`), so Power Platform connectors are optional. You can add them later to connect directly to Dataverse tables (e.g. `dvlp_ksefsetting`, `dvlp_ksefsession`).

## Local development with connections

To test Power Platform connections locally:

```powershell
# Terminal 1: Start the connection proxy
pac code run

# Terminal 2: Start the dev server
pnpm run dev
```

Open the **Local Play** URL shown by `pac code run` (in the same browser profile as your Power Platform tenant).

## Architecture

```
code-app/
├── src/                    # React application source
├── dist/                   # Build output (uploaded by pac code push)
├── power.config.json       # Generated by pac code init (metadata for SDK + CLI)
├── vite.config.ts          # Includes powerApps() plugin
├── package.json            # Includes @microsoft/power-apps SDK
└── CODE_APPS_DEPLOYMENT.md # This file
```

| Component | Purpose |
|---|---|
| `@microsoft/power-apps` | SDK — APIs for connectors, auth, platform integration |
| `@microsoft/power-apps-vite` | Vite plugin — `powerApps()` in vite.config.ts |
| `power.config.json` | Metadata — app ID, connections, environment binding |
| `pac code push` | CLI — publishes build output to Power Platform |

## Troubleshooting

| Issue | Solution |
|---|---|
| `pac: command not found` | Install: `winget install Microsoft.PowerAppsCLI`, restart terminal |
| `Code apps are not enabled` | Enable in Admin Center → Settings → Features |
| `pac code push` fails | Check `pac auth list` — ensure active auth profile |
| Build too large (>5 MB warning) | Consider code-splitting with dynamic imports |
| Connectors not working locally | Use `pac code run` for local connection proxy |
| App not visible in Power Apps | Check environment selection: `pac env who` |

## Environment variables

The app uses these environment variables (set in `.env` or `.env.production`):

| Variable | Description |
|---|---|
| `VITE_API_BASE_URL` | Azure Functions API URL |
| `VITE_AZURE_CLIENT_ID` | Entra ID App Registration client ID |
| `VITE_AZURE_TENANT_ID` | Entra ID tenant ID |
| `VITE_API_SCOPE` | API scope for token acquisition |
| `VITE_ADMIN_GROUP` | Admin role group ID (optional) |
| `VITE_USER_GROUP` | User role group ID (optional) |

> **Important**: When running as a Code App, authentication is managed by the Power Apps host. MSAL configuration is only used for standalone/local development.
