## 🇵🇱 Polski

### Dokumenty kosztowe — nowy moduł

Wersja 1.0.0 wprowadza pełny moduł obsługi dokumentów kosztowych (nie-VAT) z 7 typami dokumentów, CRUD, OCR, importem CSV/XLS, AI kategoryzacją, powiadomieniami i raportami rozkładu kosztów.

#### 7 typów dokumentów kosztowych

- ✅ Paragon (Receipt) — dowody zakupu detalicznego
- ✅ Pokwitowanie (Acknowledgment) — potwierdzenia odbioru/zapłaty
- ✅ Pro forma (ProForma) — faktury pro forma (bez VAT)
- ✅ Nota księgowa (DebitNote) — noty obciążeniowe/uznaniowe
- ✅ Rachunek (Bill) — rachunki nie-VAT
- ✅ Umowa zlecenie/o dzieło (ContractInvoice) — rozliczenia umów cywilnoprawnych
- ✅ Inne (Other) — pozostałe dokumenty kosztowe

#### API (18 endpointów)

- CRUD: lista z filtrami (typ, status, data, wyszukiwanie), tworzenie, szczegóły, aktualizacja, usuwanie
- Operacje grupowe: zatwierdzanie, odrzucanie, oznaczanie jako opłacone (batch)
- AI kategoryzacja: automatyczna klasyfikacja MPK + kategoria via Azure OpenAI
- Notatki i załączniki per dokument
- Import: podgląd CSV/XLSX, potwierdzenie, pobranie szablonu
- Raport: `GET /api/reports/cost-distribution` — rozkład kosztów wg typu, kategorii, miesiąca

#### OCR — ekstrakcja danych z dokumentów

- Analiza zdjęć/skanów dokumentów kosztowych (paragony, rachunki, pokwitowania)
- Automatyczne wypełnianie formularza na podstawie rozpoznanego tekstu
- Prompty AI dedykowane dla poszczególnych typów dokumentów

#### Powiadomienia (3 nowe typy)

- ✅ Wniosek o zatwierdzenie dokumentu kosztowego (CostDocApprovalRequested)
- ✅ Decyzja o zatwierdzeniu/odrzuceniu (CostDocApprovalDecided)
- ✅ Ostrzeżenie o przekroczeniu budżetu MPK (CostDocBudgetWarning)
- SLA check (`approval-sla-check`) rozszerzony o dokumenty kosztowe

#### Integracja z budżetami i prognozami

- `BudgetService.calculateUtilization()` uwzględnia koszty z OBYDWU tabel (faktury + dokumenty kosztowe)
- Forecast API sumuje dokumenty kosztowe w prognozach wydatków
- Dashboard budżetowy z pełną widocznością kosztów

#### Infrastruktura Dataverse

- Nowa tabela `dvlp_ksefcostdocument` z 15+ kolumnami
- Globalny OptionSet `dvlp_costdocumenttype` (7 wartości: 100000000–100000006)
- Lookup do Settings (`_dvlp_settingid_value`) + MPK Center
- Skrypt `Provision-CostDocumentSchema.ps1` — automatyczne tworzenie tabeli i kolumn

#### UI — Strona „Dokumenty kosztowe" (`/costs`)

- Lista z filtrami: typ dokumentu, status, wyszukiwanie tekstu, sortowanie (data, kwota, wystawca)
- Zaznaczanie grupowe z operacjami batch (zatwierdź / odrzuć / oznacz jako opłacone)
- Dialog tworzenia z pełnym formularzem (typ, wystawca, kwoty netto/VAT/brutto, data, kategoria, MPK, opis)
- Dialog szczegółów z sekcją AI kategoryzacji
- Dostępne w Web App (Next.js) oraz Code App (React SPA)

#### Raport rozkładu kosztów — nowa zakładka w Reports

- 4 karty KPI: dokumenty ogółem, brutto, netto, VAT
- Wykres kołowy — rozkład wg typu dokumentu
- Top 10 kategorii — pasek postępu (% wartości)
- Stacked bar chart — koszty miesięczne wg typu dokumentu
- Tabela szczegółów per typ: dokumenty, netto, VAT, brutto

#### Nowe klucze i18n

- Namespace `costs` — 38 kluczy (CRUD, formularz, tabela, statusy) w PL + EN
- Namespace `reports.costDistribution` — 19 kluczy (zakładka raportów) w PL + EN
- Nawigacja: `navigation.costs` w PL + EN

## 🇬🇧 English

### Cost Documents — New Module

Version 1.0.0 introduces a full cost document (non-VAT) management module with 7 document types, CRUD, OCR, CSV/XLS import, AI categorization, notifications, and cost distribution reports.

#### 7 Cost Document Types

- ✅ Receipt — retail purchase proofs
- ✅ Acknowledgment — payment/receipt confirmations
- ✅ ProForma — pro forma invoices (no VAT)
- ✅ Debit Note — debit/credit notes
- ✅ Bill — non-VAT bills
- ✅ Contract Invoice — civil law contract settlements
- ✅ Other — all remaining cost documents

#### API (18 Endpoints)

- CRUD: list with filters (type, status, date, search), create, get details, update, delete
- Batch operations: approve, reject, mark as paid
- AI categorization: automatic cost center + category classification via Azure OpenAI
- Notes and attachments per document
- Import: CSV/XLSX preview, confirm, download template
- Report: `GET /api/reports/cost-distribution` — cost breakdown by type, category, month

#### OCR — Document Data Extraction

- Analysis of scanned cost documents (receipts, bills, acknowledgments)
- Automatic form population from recognized text
- AI prompts dedicated to specific document types

#### Notifications (3 New Types)

- ✅ Cost document approval request (CostDocApprovalRequested)
- ✅ Approval/rejection decision (CostDocApprovalDecided)
- ✅ Cost center budget threshold warning (CostDocBudgetWarning)
- SLA check (`approval-sla-check`) extended for cost documents

#### Budget & Forecast Integration

- `BudgetService.calculateUtilization()` accounts for costs from BOTH tables (invoices + cost documents)
- Forecast API includes cost documents in spending predictions
- Budget dashboard with full cost visibility

#### Dataverse Infrastructure

- New `dvlp_ksefcostdocument` table with 15+ columns
- Global OptionSet `dvlp_costdocumenttype` (7 values: 100000000–100000006)
- Lookup to Settings (`_dvlp_settingid_value`) + Cost Center
- `Provision-CostDocumentSchema.ps1` script — automated table and column creation

#### UI — "Cost Documents" Page (`/costs`)

- List with filters: document type, status, text search, sorting (date, amount, issuer)
- Multi-select with batch operations (approve / reject / mark as paid)
- Create dialog with full form (type, issuer, net/VAT/gross amounts, date, category, cost center, description)
- Details dialog with AI categorization section
- Available in Web App (Next.js) and Code App (React SPA)

#### Cost Distribution Report — New Tab in Reports

- 4 KPI cards: total documents, gross, net, VAT
- Pie chart — distribution by document type
- Top 10 categories — progress bars (% of total value)
- Stacked bar chart — monthly costs by document type
- Details table per type: documents, net, VAT, gross

#### New i18n Keys

- `costs` namespace — 38 keys (CRUD, form, table, statuses) in PL + EN
- `reports.costDistribution` namespace — 19 keys (report tab) in PL + EN
- Navigation: `navigation.costs` in PL + EN
